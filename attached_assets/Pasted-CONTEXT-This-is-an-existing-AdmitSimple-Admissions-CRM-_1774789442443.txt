CONTEXT
This is an existing AdmitSimple Admissions CRM built with React/TypeScript (Vite), Express, Drizzle ORM, and PostgreSQL. The pipeline already has stages: New Inquiry, VOB Pending, Quote Client, Pre-Assessment, Scheduled, Admitted, Non-Viable. The Pre-Assessment stage needs its full content — three clinical forms that live inside each inquiry's detail view when it is in the Pre-Assessment stage. These forms currently exist in the live app and must be replicated exactly.

WHAT TO BUILD
Add a PreAssessmentForms component and all supporting backend routes so that when an inquiry is in the pre_assessment pipeline stage, the staff can complete three clinical forms directly inside the inquiry detail page. All form data auto-saves to the database as JSONB. Forms are protected by role-based access. A "Download Pre-Assessment Forms" button exports all completed forms as a ZIP/PDF.

DATABASE CHANGES
Add three new JSONB columns to the inquiries table (or create a separate pre_assessment_forms table with inquiry_id foreign key — use whichever matches the existing Drizzle schema pattern):
typescript// Add to inquiries table in Drizzle schema (or as separate table):
preCertFormData: jsonb('pre_cert_form_data'),           // Form 1 data
preCertFormComplete: varchar('pre_cert_form_complete', { length: 10 }).default('no'), // 'yes' or 'no'

nursingAssessmentData: jsonb('nursing_assessment_data'), // Form 2 data
nursingAssessmentComplete: varchar('nursing_assessment_complete', { length: 10 }).default('no'),

preScreeningData: jsonb('pre_screening_data'),           // Form 3 data
preScreeningComplete: varchar('pre_screening_complete', { length: 10 }).default('no'),

preAssessmentCompleted: varchar('pre_assessment_completed', { length: 10 }).default('no'), // overall
preAssessmentDate: timestamp('pre_assessment_date'),
preAssessmentNotes: text('pre_assessment_notes'),
```

Run `drizzle-kit push` to apply the migration.

## BACKEND API ROUTES

Add these six routes to the existing routes file. All require `isAuthenticated` middleware. GET routes also require `canAccessPreAssessment` middleware (staff/admin/clinical roles). Store each form's data as JSONB in the database.
```
GET  /api/inquiries/:id/pre-cert-form       — fetch Form 1 data
PUT  /api/inquiries/:id/pre-cert-form       — save Form 1 data (body: { formData, isComplete })

GET  /api/inquiries/:id/nursing-assessment  — fetch Form 2 data
PUT  /api/inquiries/:id/nursing-assessment  — save Form 2 data (body: { formData, isComplete })

GET  /api/inquiries/:id/pre-screening       — fetch Form 3 data
PUT  /api/inquiries/:id/pre-screening       — save Form 3 data (body: { formData, isComplete })

GET  /api/inquiries/:id/download-pre-assessment-forms
  — Returns a ZIP file containing all completed forms rendered as PDFs or structured JSON
  — Requires canAccessInquiries middleware
  — Returns 400 if no forms have been completed yet
  — Filename: "PreAssessment-[PatientName]-[Date].zip"
The PUT routes should:

Accept { formData: Record<string, any>, isComplete: string } in the request body
Save to the corresponding JSONB column in the inquiries table
Set isComplete field ('yes' or 'no') on the form record
Return the saved form object

The component renders three tabs (Tab 1, Tab 2, Tab 3) — one per form. Use shadcn/ui Tabs, TabsList, TabsTrigger, TabsContent.
Tab labels:

Auto-save behavior (CRITICAL): Each form uses useDebounce (1000ms) on the form data state. A useEffect watches the debounced data and fires the PUT API call automatically whenever the user stops typing. Show a subtle "Saving..." indicator while saving, "Saved" on success. Do NOT have explicit Save buttons — it all auto-saves. The only explicit action button is "Mark Complete" at the bottom of each form.
Props: { inquiryId: number }
