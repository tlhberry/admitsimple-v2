REPLIT AGENT PROMPT — AdmitSimple: HIPAA-Ready Admissions CRM for Treatment Centers

PROJECT OVERVIEW
Build a full-stack, mobile-first Admissions CRM for addiction treatment centers called AdmitSimple. This is a standalone, client-owned product (not SaaS) — each treatment center gets their own isolated deployment they fully own. It must be production-ready on first build, HIPAA-conscious in its architecture, and deeply integrated with Anthropic Claude AI throughout the workflow.
The app must look polished and professional on mobile first, work perfectly on desktop, and feel like a modern healthcare CRM. Match the following design system exactly: dark navy sidebar, clean white/light gray content area, card-based layouts, blue primary accent (#3B82F6), subtle shadows, and smooth transitions. The navigation and overall UX must remain consistent with what is described below.

TECH STACK (EXACT)

Runtime: Node.js 20+ with TypeScript
Framework: Express.js (server) + React 18 (client, Vite bundler)
Database: PostgreSQL via Drizzle ORM (use DATABASE_URL env var — Replit provides this natively)
AI: Anthropic Claude SDK (@anthropic-ai/sdk) — model claude-opus-4-5 for all AI features
Auth: Session-based auth with express-session + bcryptjs for password hashing
File handling: multer for screenshot/image uploads
Styling: Tailwind CSS + shadcn/ui components
State: TanStack Query (React Query) for all data fetching
Forms: React Hook Form + Zod validation
Charts: Recharts for all analytics/reporting charts
Date handling: date-fns
PDF: jspdf + html2canvas for report export
Environment: All secrets via .env — DATABASE_URL, ANTHROPIC_API_KEY, SESSION_SECRET

Endpoint: POST /api/ai/parse-intake
Input: multipart/form-data with field document (image file)

Receive image via multer (store in memory, max 10MB)

        type: 'image',

          type: 'base64',

      {

        Extract all patient information from this document/screenshot and return it as JSON.
        Extract these fields if present:
        - firstName, lastName
        - phone, email
        - dob (date of birth, format MM/DD/YYYY)

        - primaryDiagnosis
        - substanceHistory
        - medicalHistory
        - mentalHealthHistory

HIPAA-CONSCIOUS DESIGN REQUIREMENTS
These are architectural decisions that make the platform HIPAA-ready (note: full HIPAA compliance requires additional BAA agreements and infrastructure — this is the technical foundation):

No PHI in AI prompts by default: When sending data to Claude, always aggregate first. Use patient IDs and counts, not names/DOBs/SSNs. The exception is the screenshot parser (where Claude processes intake documents) — include a clear consent notice in the UI.
Session security: Use express-session with httpOnly: true, secure: true (in production), sameSite: 'strict'. Session timeout after 30 minutes of inactivity.
Password hashing: Always use bcryptjs with salt rounds ≥ 12.
Audit logging: Log all data access events (who accessed what, when) to a audit_logs table. Include: userId, action, resourceType, resourceId, ipAddress, timestamp.
Role-based access:

Admin: full access to all features including settings and user management
Staff: access to inquiries, patients, pipeline, reports, analytics
Clinical: read-only on inquiries, full access to patients and clinical notes


Data isolation: All queries must filter by the facility. Built for multi-tenant if needed (add facilityId column to all tables for future multi-tenant support, default to 1).
Input validation: Use Zod schemas for all API inputs. Sanitize all user inputs. Never expose raw SQL errors to the client.
AI Data Notice: Show a notice on all AI features: "Data sent to Claude AI is aggregated and de-identified where possible. Do not include SSNs or sensitive financial data in free-text fields."

### Typography
- Font: Inter (Google Fonts)
- Headings: 600 weight
- Body: 400 weight
- Small/labels: 500 weight, text-sm

### Component Standards
- **Cards**: white bg, 8px border radius, 1px border #E2E8F0, shadow-sm
- **Buttons**: Primary = blue filled, Secondary = white with border, Danger = red filled
- **Inputs**: 1px border, 6px border radius, focus ring blue
- **Badges**: Pill shape, colored background + text
- **Tables**: Striped rows, hover highlight, sticky header on scroll
- **Modals**: Center overlay (desktop), bottom sheet drawer (mobile)
- **Loading states**: Skeleton loaders (not spinners) for data
- **Empty states**: Friendly illustration or icon + descriptive text + action button

### Mobile-Specific
- Bottom navigation bar (5 tabs) fixed to viewport bottom
- All modals become full-height bottom drawers
- Cards stack vertically with 16px gap
- Touch targets minimum 44x44px
- Tables become horizontally scrollable card lists
- Floating action button (FAB) bottom-right for primary action on each page
